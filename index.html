Metal³ READ MORE

Metal³ is a set of components that enable bare metal host management with Kubernetes. It allows users to enroll bare metal machines, provision operating system images, and deploy Kubernetes clusters to them

 

The goal of this project is to provide a Kubernetes Native Application Programming Interface for managing bare metal hosts via a provisioning stack that is also running on Kubernetes

 

The mission of Metal3 is to enable Kubernetes Native Infrastructure, where infrastructure management is done just like application management, by building on already existing open source tools for bare metal host provisioning such as Ironic and integrating with the Kubernetes cluster-api project allowing Metal3 to be used as an infrastructure backend for Machine objects from the Cluster API

Metal3 Component Overview
Bare Metal Actuator Bare Metal Management Pods Bare Metal Operator
The first component is the Bare Metal Actuator, which is an implementation of the Machine Actuator interface defined by the cluster-api project. This actuator reacts to changes to Machine objects and acts as a client of the BareMetalHost custom resources managed by the Bare Metal
The operator manages a set of tools for controlling the power on the host, monitoring the host status, and provisioning images to the host. These tools run inside the pod with the operator, and do not require any configuration by the user.
Bare Metal Operator includes a Controller for a new Custom Resource, the BareMetalHost. This custom resource represents an inventory of known bare metal hosts. When a Machine is created the Bare Metal Actuator will claim one of these hosts to be provisioned as a new Kubernetes node. In response to BareMetalHost updates, the controller will perform bare metal host provisioning actions as necessary to reach the desired state.
